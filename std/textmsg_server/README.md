# `textmsg-server` -- Стандартный интерфейс плагинов для работы с сервером, использующим `textmsg`.
_Дидык Иван, 8 апреля 2026_

## Цель

Данный заголовочник предназначен для того, чтобы различные плагины сервера можно
было подключить к одному общему серверу.

## Распределение ответственности

Сервер ответственнен за:

 - Управление сокетами
 - Парсинг сообщений
 - Роутинг сообщений в зависимости от префиксов

Клиент ответственнен за:

 - Игровую логику
 - Проверку существования данного типа сообщения
 - Проверку валидности данных ему аргументов
 - Отправку сообщений о ошибке

## Нюансы реализации

Данная реализация требует C++20 ради `std::span`.

Нужно заметить, что класс `textmsg::Plugin` не имеет возможностей получения
других плагинов. Это сделано затем, чтобы в случае отсутствия менеджера плагинов
(на ранних стадиях разработки) можно было вручную насоздавать плагины в
монолитной программе.

`textmsg::Message` сделан не владеющим, чтобы на каждое сообщение не было
необходимости аллоцировать строки. Соответственно, нет гарантий, что 
после завершения `onMessage()` содержимое сообщения будет валидным.

Поскольку стандарт допускает бинарные файлы, то во всех элементах возможно
наличие нулевого символа. Поскольку это срезы строк, то наличие нулевого 
символа после срезов не гарантируется. Так что **работать со здешними срезами
через утилиты из C не стоит**.

Чтобы переводить аргументы в числа можно использовать [`std::from_chars`](https://en.cppreference.com/w/cpp/utility/from_chars.html).
