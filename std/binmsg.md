# `binmsg` -- Более компактный формат вместо `textmsg`
_Дидык Иван, 13 апреля 2026_

## Цель

Хотим бинарный формат, а не текстовый.

## Формат сообщения

Все части из `textmsg` остаются, только теперь они по другому
представлены. Также добавляется поле `u16 flags`, которое на
данный момент никак не используется.

```c
struct BinMsg {
    char prefix[8];      // Префикс, ужатый в 8 символов, лишние позиции занулены
    char type[8];        // Тип сообщения аналогично
    uint32_t seqno;      // aka ID
    uint16_t bodyLength; // Размер body[]
    uint16_t flags;      // Битовые флаги, пока стандартных нет 
    char body[];         // Данные
};
```

Аргументы кодируются так:
 - Числа (`int8/16/32/64`, `float` и `double`) записываются в таком же виде, как они были в памяти
 - Строки и блоб-ы имеют в начале длинну в виде `uint16`, затем свое содержимое.
 - Есть ещё `char64` -- тип `char[8]`, который записывается наподобие префикса/типа.

В стандартах используется синтаксис [анализатора сообщений](https://github.com/random-username-here/mipt-ded-bardak-components/blob/master/pan/README.md)
