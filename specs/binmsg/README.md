# `binmsg` -- Протокол обмена данными между клиентом и сервером

 - _Писал Дидык Иван_
 - _Идею бинарного формата предложил Кирилл Катынский_

Сначала был стандарт `textmsg`, который был, как можно догадаться, текстовым.
Его спецификацию можно увидеть в [`./textmsg.md`](./textmsg.md). Затем
была предложена бинарная версия и написан новый стандарт [`./binmsg.md`](./binmsg.md).
Он во многом основан на старом, только данные теперь бинарные.

Формат многоуровненный: здесь специфицируется нижний слой (структура сообщений,
адресация, ...). Типы сообщений и их содержание специфицируются отдельно,
в данном репозитории они в [`../../specs/proto/`](../../specs/proto/).

Имеется заголовочник для стандартизованной передачи сообщений внутри программы.
Для него настроен CMake, либа `binmsg`. Используется `std::string_view`, так что
нужен C++17.
